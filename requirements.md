# HELLO EOMJI 요구사항

## 1. 컨셉

**HELLO EOMJI**는 하루 할 일을 헬로키티와 함께 관리하고, 완료한 기록을 귀여운 공유 카드로 남기는 감정형 Todo 웹 앱이다.

앱의 중심은 딱딱한 생산성 관리가 아니라, 할 일을 추가하고 완료하는 과정이 헬로키티의 움직임과 반응으로 이어지는 작은 보상 경험이다. 사용자가 할 일과 날짜, 마감 시간을 입력하면 공원 배경 위에 헬로키티가 나타나고, 마감 전에 할 일을 완료하면 헬로키티가 사용자 쪽으로 다가온다.

헬로키티가 충분히 가까워지면 사용자는 헬로키티를 쓰다듬을 수 있다. 쓰다듬기 성공 시에는 별도 성공 이미지와 터치 중심 인터랙션을 통해 완료감을 보여준다. 반대로 마감 시간을 놓치면 헬로키티는 돌아가고, 사용자는 해당 기록을 실패 상태로 확인하거나 필요한 경우 구제할 수 있다.

완료 기록은 캘린더에서 다시 볼 수 있고, 날짜별 상세 기록은 세로형 공유 카드 이미지로 저장할 수 있다. 공유 카드는 타이틀 로고, 날짜, 할 일 카드, 키티 이미지를 포함하며, 사용자의 하루 기록을 깔끔하게 보관하거나 공유할 수 있게 한다.

이 앱의 핵심은 벌점보다 **귀여운 피드백과 기록 공유**다. 할 일을 끝내면 키티가 다가오고, 기록은 캘린더와 공유 카드로 남는다.

---

## 2. 타이틀 및 비주얼 콘셉트

앱 타이틀은 **HELLO EOMJI**로 사용한다.

타이틀은 일반 텍스트보다 로고 이미지 중심으로 보여준다.

```txt
- 메인 화면 타이틀: public/title-logo.png
- 공유 카드 타이틀: public/title-logo.png
- 공유 카드 안에는 "HELLO EOMJI" 텍스트를 직접 그리지 않는다
```

앱의 메인 비주얼은 공원 산책로 배경과 헬로키티 PNG 에셋을 중심으로 구성한다.

```txt
- 공원 배경은 이미지 에셋을 사용한다
- 데스크톱과 모바일 배경 이미지를 분리할 수 있다
- 헬로키티는 투명 PNG 프레임으로 표시한다
- 걷는 헬로키티는 2프레임 애니메이션으로 표현한다
- 실패, 공유 카드, 쓰다듬기 성공 상태는 각각 별도 키티 이미지를 사용한다
```

전체 톤은 귀엽고 단정해야 한다. 장식은 과하지 않게 두고, 할 일 입력과 캘린더 확인 같은 실제 사용 흐름이 먼저 보이게 한다.

## 3. 핵심 경험

오늘 할 일을 진행하는 기본 경험은 다음과 같다.

```txt
날짜, 할 일, 마감 시간을 입력한다
→ 추가 버튼 또는 Enter로 할 일을 추가한다
→ 공원 배경에 헬로키티가 나타난다
→ 시간이 지나며 헬로키티가 산책로 위에 머문다
→ 마감 전에 완료한다
→ 헬로키티가 사용자 쪽으로 다가온다
→ 가까워진 헬로키티를 클릭하거나 버튼으로 쓰다듬는다
→ 성공 키티 이미지와 완료 메시지로 보상을 받는다
```

실패 시 흐름은 다음과 같다.

```txt
할 일을 추가한다
→ 헬로키티가 공원에 나타난다
→ 마감 시간을 놓친다
→ 헬로키티가 뒤돌아 멀어진다
→ 해당 헬로키티는 쓰다듬을 수 없다
→ 필요하면 깜빡했어/늦게했어 같은 구제 액션으로 기록을 보정한다
```

기록을 다시 보는 경험은 다음과 같다.

```txt
캘린더 탭을 연다
→ 날짜별 완료/실패 기록을 확인한다
→ 특정 날짜를 선택한다
→ 해당 날짜의 할 일 목록을 확인한다
→ 공유 버튼을 누른다
→ 세로형 HELLO EOMJI 공유 카드 이미지를 저장한다
```

빠른 추가 경험은 다음과 같다.

```txt
자주 추가했던 할 일을 빠른 추가 목록에서 확인한다
→ 필요한 항목을 선택한다
→ 날짜와 시간을 지정한다
→ 선택 추가로 오늘 또는 지정 날짜의 할 일을 빠르게 만든다
```

## 4. 기능 요구사항

### 4.1 할 일 추가

사용자는 할 일 텍스트와 마감 시간 `시:분`을 입력할 수 있다.

할 일은 다음 방식으로 추가할 수 있다.

```txt
- 추가 버튼 클릭
- Enter 키 입력
```

아래 경우에는 할 일을 추가할 수 없다.

```txt
- 텍스트가 비어 있는 경우
- 공백만 입력한 경우
- 마감 시간이 입력되지 않은 경우
- 마감 시간이 현재 시각보다 과거인 경우
```

할 일이 정상적으로 추가되면 해당 할 일에 대응하는 헬로키티가 한 마리 생성된다.

```txt
할 일 1개 = 헬로키티 1마리
```

---

### 4.2 헬로키티 생성

할 일이 추가되면 헬로키티는 공원 배경의 산책로 먼 지점에 생성된다.

헬로키티는 처음부터 사용자에게 다가오지 않는다. 사용자가 해당 할 일을 마감 전에 완료해야 헬로키티가 다가온다.

헬로키티의 기본 상태는 다음과 같다.

```txt
- 먼 곳에 작게 표시된다
- 산책로 위에 서 있거나 앉아 있는 형태로 보인다
- 아직 쓰다듬을 수 없다
```

---

### 4.3 완료 처리

각 할 일에는 `완료` 버튼이 있다.

마감 전에 완료 버튼을 누르면 다음 동작이 발생한다.

```txt
- 할 일 상태가 completed로 변경된다
- 헬로키티가 사용자 쪽으로 다가오기 시작한다
- 완료한 할 일 수가 1 증가한다
- 돌아간 헬로키티 수는 증가하지 않는다
```

완료된 헬로키티는 CSS `scale`, `translateY`, `opacity`, `z-index`, `shadow`를 사용해 점점 화면 앞쪽으로 다가오는 것처럼 표현한다.

---

### 4.4 헬로키티 접근 표현

헬로키티는 성공 시 화면 먼 곳에서 사용자 쪽으로 다가온다.

접근 정도는 `completedAt`과 현재 시각 사이의 경과 시간으로 계산한다.

예시:

```txt
완료 직후:
작음 / 산책로 먼 곳 / 그림자 약함

접근 중:
조금 커짐 / 화면 중앙 쪽으로 이동 / 그림자 증가

접근 완료:
화면 앞쪽에 큼직하게 표시 / 머리 쓰다듬기 가능
```

헬로키티의 접근은 색상 변화가 아니라 원근감으로 표현한다.

```txt
scale 증가
translateY 증가
opacity 증가
z-index 증가
그림자 강화
```

---

### 4.5 쓰다듬기

헬로키티가 충분히 가까이 오면 `쓰다듬기` 버튼 또는 헬로키티 클릭으로 머리를 쓰다듬을 수 있다.

쓰다듬기 조건은 다음과 같다.

```txt
- 할 일 상태가 completed여야 한다
- 헬로키티 접근 진행률이 100%에 도달해야 한다
- 아직 쓰다듬지 않은 헬로키티여야 한다
```

쓰다듬기를 완료하면 다음 동작이 발생한다.

```txt
- 헬로키티 상태가 petted로 변경된다
- 쓰다듬은 헬로키티 수가 1 증가한다
- 헬로키티는 만족한 상태로 화면 앞쪽에 머문다
```

쓰다듬은 헬로키티는 간단한 텍스트나 상태 배지로 표시할 수 있다.

```txt
- 쓰다듬 완료
- 골골골...
- 만족한 헬로키티
```

---

### 4.6 실패 처리

마감 시간이 지나도 완료하지 않은 할 일은 실패 상태가 된다.

실패 상태가 되는 순간 다음 동작이 발생한다.

```txt
- 할 일 상태가 failed로 변경된다
- 헬로키티가 사용자에게 다가오지 않는다
- 헬로키티가 뒤돌아 멀어지는 연출을 한다
- 돌아간 헬로키티 수가 1 증가한다
```

실패한 헬로키티는 쓰다듬을 수 없다.

실패 상태는 헬로키티 색상 변경이 아니라 상태 배지나 텍스트로 표시한다.

```txt
- 돌아감
- 놓쳤어요
- 다음엔 제시간에 만나기
```

---

### 4.7 실패 후 완료 처리

마감 시간이 지난 뒤에는 해당 할 일을 완료해도 헬로키티 쓰다듬기 보상은 발생하지 않는다.

실패한 할 일은 다음 중 하나의 방식으로 처리한다.

```txt
- 삭제할 수 있다
- 실패 상태로 목록에 남길 수 있다
```

단, 실패 후 완료를 통해 헬로키티가 다시 다가오거나 쓰다듬기 가능 상태가 되지는 않는다.

---

### 4.8 삭제 처리

각 할 일에는 `삭제` 버튼이 있다.

삭제 버튼을 누르면 다음 동작이 발생한다.

```txt
- 해당 할 일이 목록에서 제거된다
- 해당 헬로키티도 화면에서 제거된다
- 완료한 할 일 수는 증가하지 않는다
- 쓰다듬은 헬로키티 수는 증가하지 않는다
- 돌아간 헬로키티 수는 증가하지 않는다
```

삭제는 성공도 실패도 아닌 단순 제거 처리로 본다.

---

### 4.9 상태 표시

화면 상단에는 다음 정보를 표시한다.

```txt
- 현재 활성 할 일 수
- 완료한 할 일 수
- 쓰다듬은 헬로키티 수
- 돌아간 헬로키티 수
```

활성 할 일 수는 아직 완료되거나 실패하거나 삭제되지 않은 할 일의 수를 의미한다.

---

### 4.10 빈 상태 화면

등록된 할 일이 없을 때는 빈 상태 문구를 표시한다.

예시:

```txt
공원에 아직 헬로키티가 없어요.
할 일을 추가하고 헬로키티를 만나보세요.
```

## 6. src/game.js 역할

`game.js`는 순수 함수만 담당한다.

DOM 조작, `setInterval`, `Date.now()` 직접 호출은 하지 않는다. 현재 시각은 항상 인자로 주입받는다.

포함할 함수는 다음과 같다.

```js
addTask
completeTask
failOverdueTasks
deleteTask
petCat
calculateCatApproach
calculateCatPerspective
calculateStats
canPetCat
```

---

### 6.1 addTask

할 일을 추가하는 순수 함수다.

역할:

```txt
- 텍스트 검증
- 마감 시간 검증
- 새 할 일 객체 생성
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.2 completeTask

할 일을 완료 처리하는 순수 함수다.

역할:

```txt
- active 상태의 할 일만 완료 처리
- 마감 전 완료인지 검증
- completedAt 기록
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.3 failOverdueTasks

마감 시간이 지난 할 일을 실패 상태로 변경하는 순수 함수다.

역할:

```txt
- active 상태이면서 deadlineAt이 현재 시각보다 과거인 할 일을 failed로 변경
- failedAt 기록
- 이미 실패 처리된 할 일은 중복 처리하지 않음
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.4 deleteTask

할 일을 삭제하는 순수 함수다.

역할:

```txt
- 해당 id의 할 일을 배열에서 제거
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.5 petCat

가까이 온 헬로키티를 쓰다듬은 상태로 변경하는 순수 함수다.

역할:

```txt
- completed 상태의 할 일만 처리
- 접근이 완료된 헬로키티만 처리
- 이미 쓰다듬은 헬로키티는 중복 처리하지 않음
- pettedAt 기록
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.6 calculateCatApproach

헬로키티 접근 진행률을 계산하는 순수 함수다.

반환값 예시:

```js
0   // 아직 멀리 있음
0.5 // 절반 정도 다가옴
1   // 사용자 앞까지 다가옴
```

기준:

```txt
- active 상태: 0
- completed 상태: completedAt 이후 경과 시간에 따라 0에서 1로 증가
- failed 상태: 0 또는 멀어지는 값으로 처리
- petted 상태: 1
```

---

### 6.7 calculateCatPerspective

헬로키티의 원근감 표현 값을 계산하는 순수 함수다.

반환값 예시:

```js
{
  approachRatio: 0.75,
  scale: 1.9,
  translateY: 150,
  opacity: 0.95,
  zIndex: 75,
  shadowStrength: 0.8
}
```

이 값은 `app.js`에서 CSS 스타일로 적용한다.

---

### 6.8 calculateStats

현재 할 일 목록을 기준으로 상단 통계 값을 계산한다.

반환값 예시:

```js
{
  activeCount: 2,
  completedCount: 3,
  pettedCount: 1,
  returnedCatCount: 1
}
```

---

### 6.9 canPetCat

현재 헬로키티를 쓰다듬을 수 있는지 판단한다.

조건:

```txt
- status가 completed여야 한다
- 접근 진행률이 1이어야 한다
- pettedAt이 없어야 한다
```

## 7. 브라우저 앱 스크립트 역할

현재 브라우저 앱 스크립트는 `index.html` 안에서 DOM 연결과 화면 갱신을 담당한다.

주요 역할:

```txt
- 입력값 읽기
- 추가 버튼 이벤트 연결
- Enter 키 이벤트 연결
- 완료 버튼 이벤트 연결
- 삭제 버튼 이벤트 연결
- 쓰다듬기 버튼 또는 헬로키티 클릭 이벤트 연결
- game.js 순수 함수 호출
- 화면 렌더링
- setInterval로 현재 시각 갱신
- 마감 시간이 지난 할 일 실패 처리
- 헬로키티 원근감 스타일 적용
- 공원 배경 UI 표시
- 캘린더 보기와 날짜별 기록 렌더링
- 빠른 추가 목록 렌더링
- 공유 카드 이미지 생성 및 다운로드
```

브라우저 앱 스크립트에서는 DOM을 조작해도 된다.

단, 사용자 입력을 `innerHTML`에 직접 넣으면 안 된다. 사용자 입력 출력은 `textContent`를 사용한다.

## 9. UI 반영 요구사항

### 9.1 안내 메시지

안내 메시지 영역은 기본 상태에서 공간을 차지하지 않는다.

```txt
- 메시지 텍스트가 없으면 숨김 처리한다
- 메시지가 생길 때만 해당 영역이 표시되고 높이가 생긴다
- 모바일에서 추가/빠른 추가 버튼 아래에 빈 공백을 미리 확보하지 않는다
- 안내 메시지는 표시 후 3초가 지나면 자동으로 사라진다
```

### 9.2 버튼 및 진행 바

```txt
- 캘린더 상세의 공유 버튼 문구는 "공유"로 표시한다
- 진행률 바는 그라데이션이 아니라 단일 색상으로 표시한다
```

### 9.3 헬로키티 이미지 에셋

헬로키티 이미지는 용도별 PNG 에셋을 사용한다.

```txt
- 걷기 프레임 1: public/cat-walk-a.png
- 걷기 프레임 2: public/cat-walk-b.png
- 실패/돌아감 이미지: public/cat-returning.png
- 공유 카드 키티 이미지: public/cat-summary.png
- 쓰다듬기 성공 이미지: cat-petted.png
```

걷기 프레임은 투명 배경 PNG를 사용하며, 두 프레임을 번갈아 보여 걷는 애니메이션을 만든다.

## 10. 공유 카드 요구사항

캘린더 날짜별 상세 기록은 공유 카드 이미지로 저장할 수 있다.

```txt
- 공유 카드는 세로형 1080 x 1920 캔버스로 생성한다
- 다운로드 파일명은 helloeomji-YYYY-MM-DD.png 형식을 사용한다
- 공유 카드 타이틀은 텍스트가 아니라 public/title-logo.png 이미지를 사용한다
- "오운완! 완료 n개" 문구는 표시하지 않는다
- 카드 하단의 "hello-eomji" 텍스트는 표시하지 않는다
- 공유 카드 키티는 public/cat-summary.png를 사용한다
- 공유 카드 키티는 할 일 카드보다 위 레이어에 그린다
- 공유 카드 키티는 하얀 카드 박스 안쪽 우측 하단에 배치한다
- 공유 카드 키티는 하얀 카드 박스의 오른쪽과 아래쪽에서 각각 9px 여백을 남긴다
```

할 일 카드 목록은 날짜별 할 일 개수에 따라 크기를 자동 조정한다.

```txt
- 할 일이 적은 날은 카드 높이와 글자 크기를 크게 조정해 빈 화면처럼 보이지 않게 한다
- 할 일이 많은 날은 카드 높이, 간격, 글자 크기를 줄여 세로형 공유 카드 안에 들어오게 한다
- 카드 목록이 너무 많아 영역을 넘길 경우 마지막 카드에 "외 n개 더 있어요" 요약을 표시한다
- 할 일 카드가 공유 카드 하얀 박스 밖으로 삐져나가지 않게 한다
```

## 11. tests/game.test.js 역할

Vitest로 `game.js`의 순수 함수를 테스트한다.

테스트 항목:

```txt
- 빈 텍스트는 추가할 수 없다
- 공백만 있는 텍스트는 추가할 수 없다
- 마감 시간이 없으면 추가할 수 없다
- 과거 마감 시간은 추가할 수 없다
- 정상적인 할 일은 추가된다
- 할 일 추가 시 원본 배열을 변경하지 않는다
- 마감 전 완료하면 상태가 completed가 된다
- 완료 시 completedAt이 기록된다
- 마감 후 완료는 헬로키티 보상을 발생시키지 않는다
- 마감 시간이 지나면 상태가 failed가 된다
- 실패 시 failedAt이 기록된다
- 실패 처리는 중복 적용되지 않는다
- 실패한 헬로키티는 쓰다듬을 수 없다
- 완료 직후 헬로키티 접근률은 0에 가깝다
- 시간이 지나면 헬로키티 접근률이 증가한다
- 접근이 완료되면 헬로키티를 쓰다듬을 수 있다
- 쓰다듬으면 pettedAt이 기록된다
- 이미 쓰다듬은 헬로키티는 중복 쓰다듬기 처리되지 않는다
- 삭제하면 할 일이 제거된다
- 삭제 시 원본 배열을 변경하지 않는다
- 통계 값이 올바르게 계산된다
```

## 12. 데이터 구조 예시

할 일 객체는 다음 구조를 가진다.

```js
{
  id: "task-1",
  text: "포트폴리오 정리하기",
  createdAt: 1714350000000,
  deadlineAt: 1714353600000,
  status: "active",
  completedAt: null,
  failedAt: null,
  pettedAt: null
}
```

상태값:

```txt
active     진행 중
completed  마감 전 완료, 헬로키티가 다가오는 중 또는 도착함
failed     마감 실패, 헬로키티가 돌아감
petted     쓰다듬기 완료
```

구현상 `petted`를 별도 status로 둘 수도 있고, `status: "completed"`를 유지한 채 `pettedAt` 값으로 쓰다듬기 여부를 판단해도 된다.

## 14. 제약 조건

```txt
- 핵심 로직은 game.js의 순수 함수로 구현한다
- DOM 조작은 app.js에서만 처리한다
- 시간 기반 갱신은 app.js에서 setInterval로 처리한다
- 외부 JavaScript 라이브러리는 사용하지 않는다
- Tailwind CSS는 스타일링 전용으로만 사용한다
- 빈 텍스트는 추가할 수 없다
- 공백만 있는 텍스트는 추가할 수 없다
- 마감 시각이 현재 시각보다 과거인 경우 추가할 수 없다
- 원본 배열을 변경하지 않는다
- 시간 계산은 Date 객체와 timestamp(ms)를 사용한다
- 순수 함수 안에서 Date.now()를 직접 호출하지 않는다
- 현재 시각은 인자로 주입한다
- 사용자 입력을 innerHTML에 직접 넣지 않는다
- 배경은 외부 지도 API 없이 정적 공원 일러스트 또는 이미지 에셋으로 표현한다
```
