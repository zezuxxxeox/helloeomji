# 오늘뭐하냥 요구사항

## 1. 컨셉

**오늘뭐하냥**는 마감 시간이 있는 할 일을 고양이와의 거리감으로 표현하는 게임형 Todo 웹 앱이다.

사용자가 할 일과 마감 시간을 입력하면, 한강공원 배경 위에 해당 할 일을 기다리는 고양이가 나타난다. 사용자가 마감 전에 할 일을 완료하면 고양이는 화면 먼 곳에서 사용자 쪽으로 다가오고, 충분히 가까워지면 고양이 머리를 쓰다듬을 수 있다.

반대로 마감 시간을 놓치면 고양이는 사용자에게 다가오지 않고, 실망한 듯 다시 멀어지며 화면 밖으로 돌아간다.

이 앱의 핵심은 벌점보다 **보상형 감정 인터랙션**이다. 할 일을 끝내면 고양이가 다가오고, 실패하면 고양이가 떠나는 방식으로 사용자의 행동 결과를 시각적으로 보여준다.

---

## 2. 배경 콘셉트

앱의 메인 배경은 **한강공원**으로 설정한다.

한강공원 배경은 다음 요소로 구성한다.

```txt
- 하늘
- 한강
- 잔디
- 산책로
- 멀리 보이는 다리 또는 도시 실루엣
- 벤치, 가로등 등 간단한 공원 오브젝트
```

배경은 실제 지도나 외부 이미지가 아니라, HTML과 CSS로 구성한 정적인 일러스트 스타일 배경으로 구현한다.

고양이는 한강공원 산책로의 먼 지점에서 시작해, 성공 시 산책로를 따라 화면 앞쪽으로 다가오는 것처럼 표현한다.

---

## 3. 핵심 경험

이 앱의 핵심 경험은 다음과 같다.

```txt
할 일을 추가한다
→ 고양이가 한강공원 산책로 멀리 나타난다
→ 마감 전에 할 일을 완료한다
→ 고양이가 사용자 쪽으로 다가온다
→ 고양이가 가까이 오면 머리를 쓰다듬을 수 있다
```

실패 시 흐름은 다음과 같다.

```txt
할 일을 추가한다
→ 고양이가 기다린다
→ 마감 시간을 놓친다
→ 고양이가 뒤돌아 멀어진다
→ 해당 고양이는 쓰다듬을 수 없다
```

## 4. 기능 요구사항

### 4.1 할 일 추가

사용자는 할 일 텍스트와 마감 시간 `시:분`을 입력할 수 있다.

할 일은 다음 방식으로 추가할 수 있다.

```txt
- 추가 버튼 클릭
- Enter 키 입력
```

아래 경우에는 할 일을 추가할 수 없다.

```txt
- 텍스트가 비어 있는 경우
- 공백만 입력한 경우
- 마감 시간이 입력되지 않은 경우
- 마감 시간이 현재 시각보다 과거인 경우
```

할 일이 정상적으로 추가되면 해당 할 일에 대응하는 고양이가 한 마리 생성된다.

```txt
할 일 1개 = 고양이 1마리
```

---

### 4.2 고양이 생성

할 일이 추가되면 고양이는 한강공원 산책로의 먼 지점에 생성된다.

고양이는 처음부터 사용자에게 다가오지 않는다. 사용자가 해당 할 일을 마감 전에 완료해야 고양이가 다가온다.

고양이의 기본 상태는 다음과 같다.

```txt
- 먼 곳에 작게 표시된다
- 산책로 위에 서 있거나 앉아 있는 형태로 보인다
- 아직 쓰다듬을 수 없다
```

---

### 4.3 완료 처리

각 할 일에는 `완료` 버튼이 있다.

마감 전에 완료 버튼을 누르면 다음 동작이 발생한다.

```txt
- 할 일 상태가 completed로 변경된다
- 고양이가 사용자 쪽으로 다가오기 시작한다
- 완료한 할 일 수가 1 증가한다
- 돌아간 고양이 수는 증가하지 않는다
```

완료된 고양이는 CSS `scale`, `translateY`, `opacity`, `z-index`, `shadow`를 사용해 점점 화면 앞쪽으로 다가오는 것처럼 표현한다.

---

### 4.4 고양이 접근 표현

고양이는 성공 시 화면 먼 곳에서 사용자 쪽으로 다가온다.

접근 정도는 `completedAt`과 현재 시각 사이의 경과 시간으로 계산한다.

예시:

```txt
완료 직후:
작음 / 산책로 먼 곳 / 그림자 약함

접근 중:
조금 커짐 / 화면 중앙 쪽으로 이동 / 그림자 증가

접근 완료:
화면 앞쪽에 큼직하게 표시 / 머리 쓰다듬기 가능
```

고양이의 접근은 색상 변화가 아니라 원근감으로 표현한다.

```txt
scale 증가
translateY 증가
opacity 증가
z-index 증가
그림자 강화
```

---

### 4.5 쓰다듬기

고양이가 충분히 가까이 오면 `쓰다듬기` 버튼 또는 고양이 클릭으로 머리를 쓰다듬을 수 있다.

쓰다듬기 조건은 다음과 같다.

```txt
- 할 일 상태가 completed여야 한다
- 고양이 접근 진행률이 100%에 도달해야 한다
- 아직 쓰다듬지 않은 고양이여야 한다
```

쓰다듬기를 완료하면 다음 동작이 발생한다.

```txt
- 고양이 상태가 petted로 변경된다
- 쓰다듬은 고양이 수가 1 증가한다
- 고양이는 만족한 상태로 화면 앞쪽에 머문다
```

쓰다듬은 고양이는 간단한 텍스트나 상태 배지로 표시할 수 있다.

```txt
- 쓰다듬 완료
- 골골골...
- 만족한 고양이
```

---

### 4.6 실패 처리

마감 시간이 지나도 완료하지 않은 할 일은 실패 상태가 된다.

실패 상태가 되는 순간 다음 동작이 발생한다.

```txt
- 할 일 상태가 failed로 변경된다
- 고양이가 사용자에게 다가오지 않는다
- 고양이가 뒤돌아 멀어지는 연출을 한다
- 돌아간 고양이 수가 1 증가한다
```

실패한 고양이는 쓰다듬을 수 없다.

실패 상태는 고양이 색상 변경이 아니라 상태 배지나 텍스트로 표시한다.

```txt
- 돌아감
- 놓쳤어요
- 다음엔 제시간에 만나기
```

---

### 4.7 실패 후 완료 처리

마감 시간이 지난 뒤에는 해당 할 일을 완료해도 고양이 쓰다듬기 보상은 발생하지 않는다.

실패한 할 일은 다음 중 하나의 방식으로 처리한다.

```txt
- 삭제할 수 있다
- 실패 상태로 목록에 남길 수 있다
```

단, 실패 후 완료를 통해 고양이가 다시 다가오거나 쓰다듬기 가능 상태가 되지는 않는다.

---

### 4.8 삭제 처리

각 할 일에는 `삭제` 버튼이 있다.

삭제 버튼을 누르면 다음 동작이 발생한다.

```txt
- 해당 할 일이 목록에서 제거된다
- 해당 고양이도 화면에서 제거된다
- 완료한 할 일 수는 증가하지 않는다
- 쓰다듬은 고양이 수는 증가하지 않는다
- 돌아간 고양이 수는 증가하지 않는다
```

삭제는 성공도 실패도 아닌 단순 제거 처리로 본다.

---

### 4.9 상태 표시

화면 상단에는 다음 정보를 표시한다.

```txt
- 현재 활성 할 일 수
- 완료한 할 일 수
- 쓰다듬은 고양이 수
- 돌아간 고양이 수
```

활성 할 일 수는 아직 완료되거나 실패하거나 삭제되지 않은 할 일의 수를 의미한다.

---

### 4.10 빈 상태 화면

등록된 할 일이 없을 때는 빈 상태 문구를 표시한다.

예시:

```txt
한강공원에 아직 기다리는 고양이가 없어요.
할 일을 추가하고 고양이를 만나보세요.
```

## 6. src/game.js 역할

`game.js`는 순수 함수만 담당한다.

DOM 조작, `setInterval`, `Date.now()` 직접 호출은 하지 않는다. 현재 시각은 항상 인자로 주입받는다.

포함할 함수는 다음과 같다.

```js
addTask
completeTask
failOverdueTasks
deleteTask
petCat
calculateCatApproach
calculateCatPerspective
calculateStats
canPetCat
```

---

### 6.1 addTask

할 일을 추가하는 순수 함수다.

역할:

```txt
- 텍스트 검증
- 마감 시간 검증
- 새 할 일 객체 생성
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.2 completeTask

할 일을 완료 처리하는 순수 함수다.

역할:

```txt
- active 상태의 할 일만 완료 처리
- 마감 전 완료인지 검증
- completedAt 기록
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.3 failOverdueTasks

마감 시간이 지난 할 일을 실패 상태로 변경하는 순수 함수다.

역할:

```txt
- active 상태이면서 deadlineAt이 현재 시각보다 과거인 할 일을 failed로 변경
- failedAt 기록
- 이미 실패 처리된 할 일은 중복 처리하지 않음
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.4 deleteTask

할 일을 삭제하는 순수 함수다.

역할:

```txt
- 해당 id의 할 일을 배열에서 제거
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.5 petCat

가까이 온 고양이를 쓰다듬은 상태로 변경하는 순수 함수다.

역할:

```txt
- completed 상태의 할 일만 처리
- 접근이 완료된 고양이만 처리
- 이미 쓰다듬은 고양이는 중복 처리하지 않음
- pettedAt 기록
- 기존 배열을 변경하지 않고 새 배열 반환
```

---

### 6.6 calculateCatApproach

고양이 접근 진행률을 계산하는 순수 함수다.

반환값 예시:

```js
0   // 아직 멀리 있음
0.5 // 절반 정도 다가옴
1   // 사용자 앞까지 다가옴
```

기준:

```txt
- active 상태: 0
- completed 상태: completedAt 이후 경과 시간에 따라 0에서 1로 증가
- failed 상태: 0 또는 멀어지는 값으로 처리
- petted 상태: 1
```

---

### 6.7 calculateCatPerspective

고양이의 원근감 표현 값을 계산하는 순수 함수다.

반환값 예시:

```js
{
  approachRatio: 0.75,
  scale: 1.9,
  translateY: 150,
  opacity: 0.95,
  zIndex: 75,
  shadowStrength: 0.8
}
```

이 값은 `app.js`에서 CSS 스타일로 적용한다.

---

### 6.8 calculateStats

현재 할 일 목록을 기준으로 상단 통계 값을 계산한다.

반환값 예시:

```js
{
  activeCount: 2,
  completedCount: 3,
  pettedCount: 1,
  returnedCatCount: 1
}
```

---

### 6.9 canPetCat

현재 고양이를 쓰다듬을 수 있는지 판단한다.

조건:

```txt
- status가 completed여야 한다
- 접근 진행률이 1이어야 한다
- pettedAt이 없어야 한다
```

## 7. src/app.js 역할

`app.js`는 DOM 연결과 화면 갱신을 담당한다.

주요 역할:

```txt
- 입력값 읽기
- 추가 버튼 이벤트 연결
- Enter 키 이벤트 연결
- 완료 버튼 이벤트 연결
- 삭제 버튼 이벤트 연결
- 쓰다듬기 버튼 또는 고양이 클릭 이벤트 연결
- game.js 순수 함수 호출
- 화면 렌더링
- setInterval로 현재 시각 갱신
- 마감 시간이 지난 할 일 실패 처리
- 고양이 원근감 스타일 적용
- 한강공원 배경 UI 표시
```

`app.js`에서는 DOM을 조작해도 된다.

단, 사용자 입력을 `innerHTML`에 직접 넣으면 안 된다. 사용자 입력 출력은 `textContent`를 사용한다.

## 8. index.html 역할

`index.html`은 사용자 인터페이스를 담당한다.

포함 요소:

```txt
- Tailwind CSS CDN
- 앱 제목
- 상단 통계 영역
- 한강공원 배경 영역
- 고양이 표시 영역
- 할 일 입력창
- 마감 시간 입력창
- 추가 버튼
- 할 일 목록 영역
- 빈 상태 화면
```

한강공원 배경은 다음 레이어 구조로 표현한다.

```txt
하늘 레이어
도시 실루엣 레이어
한강 레이어
잔디 레이어
산책로 레이어
고양이 레이어
```

Tailwind CSS는 스타일링 용도로만 사용한다.

## 9. tests/game.test.js 역할

Vitest로 `game.js`의 순수 함수를 테스트한다.

테스트 항목:

```txt
- 빈 텍스트는 추가할 수 없다
- 공백만 있는 텍스트는 추가할 수 없다
- 마감 시간이 없으면 추가할 수 없다
- 과거 마감 시간은 추가할 수 없다
- 정상적인 할 일은 추가된다
- 할 일 추가 시 원본 배열을 변경하지 않는다
- 마감 전 완료하면 상태가 completed가 된다
- 완료 시 completedAt이 기록된다
- 마감 후 완료는 고양이 보상을 발생시키지 않는다
- 마감 시간이 지나면 상태가 failed가 된다
- 실패 시 failedAt이 기록된다
- 실패 처리는 중복 적용되지 않는다
- 실패한 고양이는 쓰다듬을 수 없다
- 완료 직후 고양이 접근률은 0에 가깝다
- 시간이 지나면 고양이 접근률이 증가한다
- 접근이 완료되면 고양이를 쓰다듬을 수 있다
- 쓰다듬으면 pettedAt이 기록된다
- 이미 쓰다듬은 고양이는 중복 쓰다듬기 처리되지 않는다
- 삭제하면 할 일이 제거된다
- 삭제 시 원본 배열을 변경하지 않는다
- 통계 값이 올바르게 계산된다
```

## 10. 데이터 구조 예시

할 일 객체는 다음 구조를 가진다.

```js
{
  id: "task-1",
  text: "포트폴리오 정리하기",
  createdAt: 1714350000000,
  deadlineAt: 1714353600000,
  status: "active",
  completedAt: null,
  failedAt: null,
  pettedAt: null
}
```

상태값:

```txt
active     진행 중
completed  마감 전 완료, 고양이가 다가오는 중 또는 도착함
failed     마감 실패, 고양이가 돌아감
petted     쓰다듬기 완료
```

구현상 `petted`를 별도 status로 둘 수도 있고, `status: "completed"`를 유지한 채 `pettedAt` 값으로 쓰다듬기 여부를 판단해도 된다.

## 12. 제약 조건

```txt
- 핵심 로직은 game.js의 순수 함수로 구현한다
- DOM 조작은 app.js에서만 처리한다
- 시간 기반 갱신은 app.js에서 setInterval로 처리한다
- 외부 JavaScript 라이브러리는 사용하지 않는다
- Tailwind CSS는 스타일링 전용으로만 사용한다
- 빈 텍스트는 추가할 수 없다
- 공백만 있는 텍스트는 추가할 수 없다
- 마감 시각이 현재 시각보다 과거인 경우 추가할 수 없다
- 원본 배열을 변경하지 않는다
- 시간 계산은 Date 객체와 timestamp(ms)를 사용한다
- 순수 함수 안에서 Date.now()를 직접 호출하지 않는다
- 현재 시각은 인자로 주입한다
- 사용자 입력을 innerHTML에 직접 넣지 않는다
- 배경은 외부 지도 API 없이 정적 한강공원 일러스트로 표현한다
```
